<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| faculty browser → own attendance record | The start action writes the session's own presence facts. |
| pasted URL → rendered link a Checker will click | The link is user-supplied and another user follows it. |
| session pk in POST → which record is written | The pk is client-supplied. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-47 | Elevation of Privilege | starting another faculty member's session | high | mitigate | Ownership re-gated server-side on the re-fetched row, never from the rendered list; asserted by a forged-pk test. |
| T-07-48 | Tampering | a non-Teams link rendered to a Checker as a meeting to join | high | mitigate | https required plus host-suffix allow-list of Teams domains; substring matching explicitly rejected and covered by a test. |
| T-07-49 | Spoofing | self-start passing as Checker verification | high | mitigate | `verified_by_checker` untouched, no `CheckerValidation` written, and a distinct `ONLINE_SELF` check-in method so the two acts stay separable in the data (D-01/D-02). |
| T-07-50 | Tampering | a self-start used to escape an Absent marking | medium | mitigate | Start refused unless status is SCHEDULED and refused past grace using the shared `is_no_show_past_grace` predicate, imported read-only. |
| T-07-51 | Repudiation | silent overwrite of a meeting link | medium | mitigate | `session.teams_link_set` AuditLog carrying the previous value. |
| T-07-52 | Tampering | drift between this surface's grace rule and the scan/sweep rule | high | mitigate | The shared predicate is called, not copied; `web/scan.py` and `scheduling/resolver.py` are not modified (D-09). |
</threat_model>
