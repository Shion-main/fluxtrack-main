<task type="auto">
  <name>Task 2: Faculty attendance history view and page</name>
  <files>web/faculty.py, web/urls.py, templates/faculty/history.html</files>
  <action>
Add `history` to `web/faculty.py` behind `faculty_required` and `@require_http_methods(["GET"])`. `web/hr.py:178` `attendance` is the closest existing surface by every axis and is the model to copy — but with one structural difference that is the whole point of this view.

HR's version is cross-department and takes a faculty filter from the querystring. This one is HARD-SCOPED: the queryset filters `faculty=request.user` and there is no faculty parameter at all. Not a defaulted one, not an ignored one — the surface must not accept the concept. That is the IDOR re-gate rule this project states at `web/faculty.py:447`, and it is why this is a separate view rather than HR's with a different template.

Build the queryset following `_filtered_sessions` at `web/hr.py:86`:
  - `select_related("schedule", "schedule__term", "room")` for everything rendered; the faculty is known, so it does not need joining.
  - Annotate `is_verified` with `Exists()` over `CheckerValidation` filtered to `action=ValidationAction.VERIFIED`. Use the annotation, NOT the `Session.verified_by_checker` property: the property issues a subquery per object, which 06-07 found fatal inside a streaming iterator on MSSQL and which is an N+1 even here.
  - Annotate `is_flagged` with a second `Exists()` over `CheckerValidation` filtered to the two flag actions, `FLAG_IDENTITY_MISMATCH` and `FLAG_NOT_PRESENT`. `VERIFIED_EMPTY` is not a flag against the faculty member and must not be counted as one.
  - Order newest-first by date then scheduled_start, matching HR, so the page cap shows the most recent sessions rather than the oldest.
  - Offer date-range and term filters parsed exactly as HR parses them: FK-id filters applied only when the raw value is a clean integer, dates via `parse_date` with an invalid date degrading to a friendly inline note and the bound simply dropped — a 200 with a notice, never a 500. Key filters on the FK id and `date__range` only. Never build a `pk__in` list: this project has been bitten by the MSSQL 2100-parameter limit before (the 04.1-04 `reset_term` incident) and 06-07 avoided it the same way.
  - Paginate with `paginate(request, qs, per_page=...)` from `web/pagination.py`, which carries active filters into every page link, and render with `{% include "_pager.html" %}`.

For the rows the page will render, attach `status_label(s.status)` from the shared helper, and fetch the flag detail for the flagged rows. Fetch that detail with ONE additional query over the page's session ids rather than per row — the page is capped, so a bounded id list here is safe, but materialize the page's object list with `list()` before issuing it (MSSQL runs with MARS off, so a follow-up query issued while an outer cursor is open raises HY010; the guards in `scheduling/jobs.py` and `web/ifo.py:131` are the precedent).

`templates/faculty/history.html` extends `base.html` and uses the navy `.ft-*` shell, matching `templates/faculty/schedule.html`: `.ft-app`, `.ft-top`, `.ft-page`, `.ft-greet`. The table uses `.tbl` inside `.table-wrap` from `static/css/app.css`. Status renders as a `.ft-pill--*` chip from the existing family (`--active`, `--absent`, `--upcoming`, `--done`), and a checker flag renders as a chip in the `.ft-pill--absent` family showing the flag reason — each with an icon and a text label so colour is never the only signal.

There is NO contest, dispute, appeal or "request review" control anywhere on this page (D-15). If the page needs to say anything about disagreement, it says to contact HR. Do not add a form, a button, or a link that implies the flag can be challenged in the app.

Filter controls use `.ft-form`/`.ft-field`/`.ft-control`; empty and no-results states are explicit messages, not a bare table. No border-left accent stripes; colours from `static/css/tokens.css`. No CSV export — none is requested by FAC-11.

URL name `faculty_history`, in the Faculty block of the flat `web/urls.py` list. Link it from the faculty navigation.
