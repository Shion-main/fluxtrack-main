Release action. Add `session_release` to `web/ifo.py` behind `ifo_required` and `@require_http_methods(["POST"])`. Follow the `modality_withdraw` shape at `web/faculty.py:441` — POST-only, re-fetch by pk, delegate to the domain service, render the refusal at 400. Re-fetch the session with `get_object_or_404` by pk, then re-gate server-side before acting: refuse when the session is not currently holding the room (already released, i.e. `room_released_at` is not NULL) with a plain message saying so, and refuse when the session is in a status that never held the room. That re-gate is the control; the button the operator clicked is a snapshot that may be minutes stale.

Call `ops.occupancy.release_room(session, actor=request.user)`. Do NOT write an AuditLog in the view — `release_room` already writes `session.room_released` with the actor and the release instant, and a second row would double-count every release. This is the documented exception to the write-an-audit-per-action convention, and the view docstring must say so explicitly so a later reviewer does not "fix" the missing audit.

Do NOT touch the `RoomConflictFlag` row. D-11: the flag closes because the cause is fixed. `detect_room_conflicts` already auto-resolves open flags whose conflict key is no longer present, and adding a manual close here would create a second, competing resolution path that could mark a flag resolved while the conflict persists.

Open-conflicts page. Add `conflicts` to `web/ifo.py` behind `ifo_required` and `@require_http_methods(["GET"])`. Query open flags — `RoomConflictFlag.objects.filter(resolved_at__isnull=True)` with `select_related("room__floor__building")` — and for each, load the sessions currently contending for that room (ACTIVE with `room_released_at` NULL), `select_related` on schedule, faculty and room so the table has no N+1. Materialize the flag list with `list()` before issuing the per-flag session queries: MSSQL/pyodbc runs with MARS off, so a follow-up query issued while the outer SELECT cursor is open raises HY010, exactly as the guards in `scheduling/jobs.py` and `web/ifo.py:131` describe. Paginate with `paginate(request, qs, per_page=...)` from `web/pagination.py` if the flag list can grow; render the pager with `{% include "_pager.html" %}`.

URL names `ifo_conflicts` and `ifo_session_release`, in the existing IFO block of the flat `web/urls.py` list.

`templates/ifo/conflicts.html` extends `ifo/_console.html`. It renders one group per open flag: the room, the building and floor, when it was detected, and a table of the contending sessions (course, section, faculty, scheduled window, status) built with `.tbl` inside `.table-wrap` from `static/css/app.css`. Each session row carries a Release control that POSTs to `ifo_session_release`. The page must explain, in plain words, what release does and what happens next: releasing the session that should not be holding the room frees it, and the flag closes automatically on the next sweep run rather than needing to be dismissed. Add the new page to the IFO console navigation in `templates/ifo/_console.html` if the nav is where operators would look for it, matching the existing `cns__link` markup and active-state derivation from `url_name`. Empty state: a clear "no open room conflicts" message, not a bare empty table.

In `templates/ifo/_room_panel.html`, add the same Release control to the panel's "right now" section when the current session is holding the room, so IFO can act from the board without navigating away. The panel is loaded into a polled board's panel target — target the release POST at a small result region inside the panel, not at the polled board container, so a refusal message is not wiped by the next poll tick.

Use the console vocabulary throughout (Franken UI `uk-*` plus Tailwind utilities as in `templates/ifo/_assignment_form.html`), not the `.ft-*` navy vocabulary, which belongs to the Faculty and Guard shells. Colour is never the only signal — the conflict indicator carries an icon and a text label. No border-left accent stripes.
