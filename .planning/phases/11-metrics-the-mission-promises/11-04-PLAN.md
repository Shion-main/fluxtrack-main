## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Room name (imported) -> CSV cell | Imported room codes/names reaching a spreadsheet (formula-injection risk). |
| Request -> export filename | A request-derived filename could enable path traversal / header injection. |
| Any authenticated user -> per-room CSV + ghost list | Institution-wide facilities data, IFO-only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-11-10 | Tampering | CSV/formula injection in the per-room export | high | mitigate | `csv_safe` on every text cell (code/name/building); a formula-name neutralization test. |
| T-11-11 | Tampering | Path traversal / header injection via export filename | medium | mitigate | Server-built filename only (`utilization-{start}.csv`); never request-derived (mirror web/hr.py:243). |
| T-11-12 | Elevation of Privilege | Non-IFO access to per-room CSV / ghost list | high | mitigate | `@ifo_required @require_http_methods(["GET"])` on `utilization_csv`; 403/405 tests. |
| T-11-13 | Information Disclosure | Raising ghost aggregate leaks an exception / blanks the page | medium | mitigate | `safe_card(ghost_rooms, ...)` per-card isolation; generic message; isolation test. |
</threat_model>
