<threat_model>
## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| client -> HR surface | Any authenticated user could try /hr/*; only Role.HR_ADMIN (or superuser) may |
| session data -> payroll CSV opened in Excel | User-controlled faculty names could become formulas |
| filtered queryset -> streamed response | A large export must not exhaust memory; a streamed cursor must not run concurrently with a write |

## STRIDE Threat Register
| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-14 | Elevation of Privilege | /hr/attendance, /hr/attendance.csv | high | mitigate | Both views wrapped in hr_required (login_required + Role.HR_ADMIN, superuser bypass); non-HR 403 (tested). |
| T-06-02 | Tampering | attendance_csv text cells | high | mitigate | Reuses scheduling/report_render.csv_safe so no faculty-name cell becomes an Excel formula (tested). |
| T-06-03 | Denial of Service | full-term CSV export | medium | mitigate | StreamingHttpResponse + queryset.iterator() bounds memory; the on-screen list is page-capped. |
| T-06-15 | Tampering | streaming generator + MSSQL cursor | medium | mitigate | No database write is performed inside the streaming generator (HY010/cursor-open trap avoided); export is read-only. |
| T-06-16 | Input Validation | filter GET params | low | mitigate | parse_date-validated; invalid input yields a friendly empty result, never a 500; filters use FK + date__range (no pk__in list). |
</threat_model>
