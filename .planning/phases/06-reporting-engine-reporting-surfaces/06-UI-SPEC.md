# Phase 6 — UI Design Contract

> Visual and interaction contract for the reporting surfaces (Weekly Consolidated Report, faculty scorecard, IFO-09 dashboard, Dean dashboard DEAN-04, HR attendance view). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Grounding note:** No CONTEXT.md exists for this phase (discuss-phase skipped) and this contract was authored non-interactively. It therefore **detects and extends the existing FluxTrack design system** rather than proposing anything greenfield. Every token, component, and idiom below is derived from files already in the repo (`templates/base.html`, `static/faculty/faculty.css`, `templates/ifo/*.html`, `templates/dean/queue.html`, `templates/notifications/_rows.html`). Genuinely novel decisions are flagged in **Assumptions** at the end for the planner to lock.

| lg | 24px | `mb-6` | Below intro paragraph / section spacing |

## Typography

Franken `uk-font-sm` base; standard shell uses `uk-h2/h3/h4`, `font-semibold`, `uk-text-small`, `uk-text-muted`.

| Role | Size | Weight | Line Height | Franken hook |
|------|------|--------|-------------|--------------|
| Body | 14px | 400 | 1.5 | default body / table cells |
| Label | 12px | 600 | 1.4 | `uk-text-small` — metric captions, table meta, timestamps, status pills |
| Heading | 24px | 600 | 1.25 | `uk-h3` — page titles on report/scorecard/HR; section headers `uk-h4` (20px/600) |
| Display | 30px | 600 | 1.2 | `uk-h2` — big metric numbers on summary cards; IFO dashboard title |

Weights declared: **Regular 400** (body) + **Semibold 600** (all emphasis, headings, metrics, labels). Numeric metric values are set in `uk-h2`/`uk-h3` semibold; captions in `uk-text-small uk-text-muted`.

## Color

60/30/10 on the neutral standard shell. Brand navy `#001c43` (the app `theme-color`, and the ReportLab PDF header color in RESEARCH) is the single accent.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (`bg-background`) | Page background, card interiors, table body |
| Secondary (30%) | `bg-muted` / `border-border` / `#f4f6fa` zebra | Card borders, table dividers, zebra rows, filter-bar surface, hover (`hover:bg-muted`) |
| Accent (10%) | **`#001c43`** (brand navy → `uk-btn-primary`, `uk-label-primary`) | Primary export/generate button, table header fill, active/selected filter, the single highlighted KPI (Attendance %), drill-down chevrons, "polling"/"latest" labels |
| Destructive | `#ba1a1a` (`--red`, `uk-btn-destructive`) | Not used for actions this phase (read-only) — reserved only for the "poor attendance" status tint and any error emphasis |

Accent reserved for: **primary export/generate CTA, the report table header row, the active date-range/term filter state, the Attendance-% headline metric, and drill-down affordances.** Never applied to every card, every link, or body text.

### Semantic status colors (existing global `:root` tokens from `faculty.css`, available shell-wide)

Used for attendance status badges and the KPI tint — reuse the existing CSS variables, do not invent new hexes:

| Meaning | Text | Tint bg | Token |

| Held / present / good attendance | `#0f7a4d` | `#dcfce7` | `--green` / `--green-tint` |
| Late / warning attendance | `#b45309` | `#fef1d6` | `--amber` / `--amber-tint` |
| Absent / poor attendance | `#ba1a1a` | `#fbe1e1` | `--red` / `--red-tint` |

## Reusable Interaction Patterns

### Pattern A — Date-range / Term selector (recurring control; specify once)

A single GET filter bar reused by IFO-09, the faculty scorecard (period), the Dean dashboard, and the HR list. Server-rendered `<form>`; on submit, htmx swaps the results panel (`hx-get` + `hx-target="#report-panel" hx-swap="innerHTML"`), falling back to a full GET navigation without JS.

- Layout: `uk-card uk-card-body` filter bar, `flex flex-wrap items-end gap-3`.
- Controls: **From** (`<input type="date" class="uk-input">`), **To** (`<input type="date" class="uk-input">`), **Term** (`<select class="uk-select">` populated from `AcademicTerm`, active term preselected), and for HR additionally **Faculty** and **Department** (`uk-select` / searchable `uk-input` with `search` icon).
- Apply button: `uk-btn uk-btn-primary uk-btn-sm` labelled **"Apply"**; a `uk-btn uk-btn-ghost uk-btn-sm` **"Reset"** clears to the default range.
- Active-filter state uses the navy accent (`uk-btn-primary`), inactive stays secondary/ghost.
- Invalid input (bad date) degrades to a friendly empty result, never a 500 (mirrors `ifo.assignment_create` validation; RESEARCH V5).

### Pattern B — Summary (KPI) card

`uk-card uk-card-body` containing: caption (`uk-text-small uk-text-muted`, e.g. "Attendance %") on top, big value (`uk-h2` semibold, navy for the highlighted Attendance-% card, foreground otherwise), optional sub-line (`uk-text-small uk-text-muted`, e.g. "of 214 scheduled"). Cards sit in the `grid gap-3 sm:grid-cols-2 lg:grid-cols-3` grid. DEAN-04 = Faculty / Sessions / Absences / Attendance %.

### Pattern C — Export affordances (download, not browser-print)

**Critical:** the PDF is generated server-side by ReportLab and delivered as a file download (per RESEARCH) — it is **NOT** `window.print()` of the HTML. Export controls are therefore plain download **anchors** to GET endpoints, not print buttons:

```
<a href="/dean/reports/{week}/export.csv" class="uk-btn uk-btn-secondary uk-btn-sm" download>
  <uk-icon icon="download" class="size-4"></uk-icon> Export CSV</a>
<a href="/dean/reports/{week}/export.pdf" class="uk-btn uk-btn-primary uk-btn-sm" download>
  <uk-icon icon="file-text" class="size-4"></uk-icon> Download PDF</a>
```

Placed top-right of the report header (mirrors the `ml-auto` action button in `ifo/rooms.html`). No client-side print CSS, no `@media print` reliance.

### Pattern D — Drill-down link

Faculty name in a report/dashboard row is a link to that faculty's scorecard, styled as the table cell text with a trailing `chevron-right` (mirrors the whole-card `<a>` link idiom in `ifo/rooms.html`). Scorecard opens as a **full page** (`/…/scorecard/{faculty_id}?from=…&to=…`), not a modal — consistent with every other detail view in the app (assumption A-DRILL: page over modal).

### Pattern E — RPT-05 per-card graceful degradation (MUST-HAVE — the load-bearing interaction contract)

When one aggregate raises, **that card alone** shows an inline error state while every other card and the surrounding page render normally (the `safe_card()` `(value, error)` pair from RESEARCH). This is a hard requirement, not decoration.

**Error-card visual:**
- Same `uk-card uk-card-body` footprint as a healthy card (no layout shift / no page blanking).
- Muted, non-alarming treatment: `border-border`, subtle `--red-tint` (`#fbe1e1`) left accent or icon only — **not** a full-red card.
- Leading `<uk-icon icon="triangle-alert">` in `--amber` (`#b45309`).
- Heading (`uk-text-small font-medium`): **"Couldn't load this section"**
- Body (`uk-text-small uk-text-muted`): **"This metric failed to load. The rest of the page is unaffected — refresh to try again."**
- **Generic copy only** — never render the raw exception string (security V7 information-disclosure control).
- If the card supports htmx, offer an inline `uk-btn uk-btn-ghost uk-btn-sm` **"Retry"** that re-fetches just that card's partial.

**Error-card markup contract (for the template):**
```
{% if card.1 %}  {# card.1 = error message, truthy on failure #}
  <div class="uk-card uk-card-body border-border">
    <div class="flex items-start gap-2">
      <uk-icon icon="triangle-alert" class="size-4" style="color:var(--amber)"></uk-icon>
      <div>
        <div class="uk-text-small font-medium">Couldn't load this section</div>
        <p class="uk-text-small uk-text-muted mt-1">This metric failed to load. The rest of the page is unaffected — refresh to try again.</p>
      </div>
    </div>
  </div>
{% else %}
  {# render card.0 normally #}
{% endif %}
```

### Pattern F — Empty state

When a range/department yields no sessions: a single `uk-card uk-card-body` with `uk-text-muted` copy (mirrors the `{% empty %}` block in `ifo/rooms.html` and `notifications/_rows.html`). Not an error — a calm "nothing here yet" with a next step.

## Per-Surface Contract Summary

| Surface | Container | Primary content | Primary CTA | States needed |
|---------|-----------|-----------------|-------------|---------------|
| Weekly Consolidated Report (RPT-01/03) | `max-w-4xl` | `uk-table` one row/faculty (scheduled, held, absent, attendance %, checker-verified) + itemized absences; per-dept | **Download PDF** (+ Export CSV secondary) | empty, error-card, loading |
| Faculty scorecard (RPT-04) | `max-w-4xl` | KPI cards (scheduled vs held, %, absences, early-ends) + modality-breakdown; period selector | Export CSV | empty, error-card |
| IFO-09 dashboard | `max-w-5xl` | Summary card grid over selectable range + scorecard drill-down | (dashboard — export lives on report) | empty, error-card, per-card isolation |
| Dean dashboard (DEAN-04) | `max-w-2xl` | 4 dept-scoped cards (Faculty, Sessions, Absences, Attendance %) + latest-weekly-report card | **Download PDF** (dept report) | read-only, empty, error-card |
| HR attendance view (HR-01/02/03) | `max-w-5xl` | Filter bar (faculty/dept/date/term) → `uk-table` session-level list | **Export CSV** | empty, error, loading, no-results |

Read-only enforcement (DEAN-01): Dean and HR reporting surfaces have **no write controls** — no destructive buttons, no edit affordances. Scoping is server-side (Dean sees only their department).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA — Weekly report / Dean report | **Download PDF** (secondary: **Export CSV**) |
| Primary CTA — HR attendance | **Export CSV** |
| Filter apply / reset | **Apply** / **Reset** |
| Drill-down | (faculty name link) + `chevron-right`; scorecard header: **"Attendance scorecard — {Faculty name}"** |
| Empty state heading | **No attendance in this range** |
| Empty state body | **No sessions were scheduled for the selected department and dates. Adjust the range or term above.** |
| HR no-results | **No sessions match these filters. Clear a filter or widen the date range.** |
| Error state (RPT-05, per card) | **Couldn't load this section** — *This metric failed to load. The rest of the page is unaffected — refresh to try again.* (generic; never raw exception text) |
| Report-ready notification (RPT-02, `WEEKLY_REPORT_READY`) | Title: **Weekly attendance report ready** · Body: **{DEPT} · week of {date}** |
| Destructive confirmation | **None — Phase 6 is read-only reporting (DEAN-01).** No delete/edit/destructive actions exist on any reporting surface; no confirmation dialogs required. |

## Assumptions (novel decisions — planner to confirm)

| # | Assumption | Impact if wrong |
|---|-----------|-----------------|
| A-COLOR | Attendance-% badge thresholds: ≥90 green · 75–89 amber · <75 red, rendered via existing `--green/--amber/--red` tokens | Thresholds are a policy choice; adjust the cutoffs or drop the color coding. Low risk — tokens already exist. |
| A-DRILL | Faculty scorecard drill-down opens as a **full page**, not a modal (consistent with every other detail view) | If a modal is preferred, swap to a Franken `uk-modal`. Low risk. |
| A-TABLE | Weekly report + HR list use `uk-table uk-table-divider uk-table-small` (dense) rather than the card-list idiom | If card-list is preferred for mobile, wrap rows responsively. The report is inherently tabular (one row/faculty) so a table is the right density. |
| A-POLL | Reporting dashboards are **point-in-time** (refresh on filter Apply via htmx swap), not continuously polled like `ifo/live.html` | If IFO-09 should live-poll, add `hx-trigger="every {{ poll_ms }}ms"`. Reporting data changes slowly; polling is optional. |
| A-EXPORT-SCOPE | Report export buttons are per-department downloads; an "All departments" roll-up file is offered to IFO only if the IFO view wants it (RESEARCH Open Q2) | Add an "All" export anchor on the IFO surface if required. |
