- A1: "pure aggregate" means side-effect-free and deterministic given inputs + DB state (resolver discipline: no writes, no notify(), no now() inside) — NOT literally ORM-free. Attendance is fundamentally a DB count.
- A2: RPT-05 card failures are surfaced in-card via a generic message + a `logging` line; NOT written to AuditLog (a read failure is not a domain state change). safe_card returns a generic message, never raw exception text.
- A5: "held" = status ACTIVE or COMPLETED; "scheduled" = all Session rows in range whose schedule.status is ACTIVE. An optional `as_of` date arg clamps the denominator to date <= as_of so a future not-yet-missed session does not lower attendance %. Archived-schedule sessions are excluded.
</assumptions>
