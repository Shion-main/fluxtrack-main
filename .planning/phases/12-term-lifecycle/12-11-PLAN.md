## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Django Admin staff POST -> term-owned ORM rows | Staff/superuser Admin requests can bypass normal IFO views and service entry points. |
| Admin bulk delete -> historical Schedule/Session/report graph | The delete-selected action can remove many rows and cascade before a per-object guard runs. |

## STRIDE Threat Register (ASVS L1; high blocks)

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-12-04A | Tampering | Admin save_model for archived rows | high | mitigate | Shared Admin guard derives owner term and calls require_writable_term before save. |
| T-12-04B | Tampering/Data loss | Admin delete_model/delete_queryset | critical | mitigate | Deny AcademicTerm deletion and validate all term-owned queryset objects before any bulk delete. |
| T-12-05A | Elevation | Raw superuser Admin bypass | high | mitigate | Superuser cannot raw-mutate archived rows; break-glass correction uses lifecycle reopen and audited services. |
| T-12-02A | Repudiation | Admin correction of archived history | high | mitigate | Refused Admin attempts leave history unchanged; successful correction requires reopen and existing audited write paths. |
</threat_model>
