<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| IFO GET query -> report scope | Term/date inputs are untrusted and can be missing, stale, invalid, or outside selected-term bounds. |
| IFO links -> drill-down/export/download | Dropped query values silently switch to ACTIVE or all-term reports. |
| Stored report id -> default_storage artifact | A valid pk can belong to another term unless lookup includes selected term. |

## STRIDE Threat Register (ASVS L1; high blocks)

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-12-06I | Information Disclosure | IFO dropped term links | high | mitigate | Shared ReportScope, normalized scope_query, and link assertions for dashboard/utilization/scorecard/weekly paths. |
| T-12-12I | Information Disclosure | IFO cross-term aggregates | high | mitigate | Required `term=scope.term` in every aggregate/export call and same-date leakage fixtures. |
| T-12-09I | Tampering/Repudiation | stored report pk mismatch | high | mitigate | Download lookup by pk plus selected term; foreign-term pk returns 404. |
| T-12-10I | Tampering | out-of-range report dates | medium | mitigate | Central date clamping from ReportScope and bounded dates in every propagated link. |
</threat_model>
