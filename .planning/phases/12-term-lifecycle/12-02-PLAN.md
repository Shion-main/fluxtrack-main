<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Scheduler/CLI -&gt; materializer | Optional term input and job timing must not select Draft/Archived history. |
| Activation -&gt; materializer/SQL Server | Many session writes and state/audit must commit as one readiness transition. |

## STRIDE Threat Register (ASVS L1; high blocks)

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-12-01 | Tampering | simultaneous activation | high | mitigate | Lock/recheck plus database filtered uniqueness; losing transaction rolls back and gets controlled error. |
| T-12-03 | Tampering/DoS | partial bulk materialization | high | mitigate | One outer transaction, list-before-write, injected exception rollback tests, term-bound window. |
| T-12-04 | Tampering | materializing archived history | high | mitigate | Normal service/command reject ARCHIVED and Draft; only activation owns the Draft exception. |
| T-12-07 | Tampering | implicit/hardcoded command target | high | mitigate | Authoritative ACTIVE resolver or exact ACTIVE `--term`; no default name/first-row behavior. |
</threat_model>
