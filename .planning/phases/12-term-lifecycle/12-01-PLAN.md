<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP/command caller -&gt; lifecycle service | Actor, confirmation, reason, and acknowledgements are untrusted and may be stale. |
| Lifecycle service -&gt; SQL Server | Concurrent transactions can race state checks; the database is the final single-ACTIVE backstop. |
| Legacy schema -&gt; migrated schema | Production-shaped rows may be ambiguous or corrupt and must not be guessed or deleted. |

## STRIDE Threat Register (ASVS L1; high blocks)

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-12-01 | Tampering | concurrent ACTIVE writes | high | mitigate | Filtered SQL uniqueness plus service transaction/lock; constraint tests on MSSQL. |
| T-12-02 | Elevation/Repudiation | lifecycle writes | high | mitigate | Service-level IFO/superuser authorization, exact confirmation, reason, and atomic AuditLog. |
| T-12-03 | Tampering | stale preflight/partial transition | high | mitigate | Re-fetch and recompute under `select_for_update()` in one atomic block; rollback tests. |
| T-12-08 | Tampering/DoS | legacy data migration | high | mitigate | Fail-loud preflight for ambiguity/corruption, row-count rehearsal, no deletes, SQL review. |
</threat_model>
