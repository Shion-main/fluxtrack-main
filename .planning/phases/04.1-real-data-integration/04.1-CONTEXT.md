## Implementation Decisions (LOCKED)

### D1 ‚Äî Read .xlsx directly
Extend `import_offerings` to read `.xlsx` via a stdlib `zipfile` + `xml.etree` reader (sharedStrings + sheet grid). **No new pip dependency** (openpyxl/pandas). Pattern proven this session. Keep CSV support or replace ‚Äî planner's call, but xlsx is the real source now.

### D2 ‚Äî Stop dropping online + gym meetings
Remove the "skip V-rooms" and the implicit gym drop (the old `parse_room` requires the 2nd char to be a digit, which fails `GYM2`). Target after fix: ~1,100 sections / ~2,021 schedules / ~200 instructors.

### D3 ‚Äî Full room master first, then attach schedules
Load all 114 master rooms as named `Room` rows with real capacities BEFORE importing offerings. Offerings attach to existing rooms; still auto-create the 7 scheduled-but-not-in-master rooms on the fly. Virtual V-rooms (92) created as non-scannable online placeholders.

### D4 ‚Äî Explicit prefix‚Üíbuilding map (replaces "first letter=building, second char=floor")
`R‚ÜíAcademic Building`, `A‚ÜíAdmin Building`, `GYM‚ÜíGym`, `V‚ÜíOnline (virtual)`, and `P` / `U` / digit-only / typo codes ‚Üí an **"Unassigned"** building, **flagged loudly** in the import report. Never silently drop an unknown room. Floor derivation must tolerate `GYM2`, `A410-A`, etc.

### D5 ‚Äî Per-meeting modality (stamped by room, not by course Mode)
Physical room ‚Üí keep course-level mode (f2f/blended), scannable. Virtual V-room ‚Üí force `modality=online`, checker-verified. A blended course yields scannable physical-day rows AND online-verified virtual-day rows.

### D6 ‚Äî Term anchored to TODAY
Keep the importer's existing behavior of centering the `AcademicTerm` window on now (NOT the historically-accurate past 2T dates) so faculty can check in live.

### D7 ‚Äî Instructor dedup: email, then normalized name
Dedup emailed instructors by email (190). Dedup the 57 blank-email rows by a **normalized name key** (case/comma/spacing-insensitive; drop single-letter middle initials) so each of the ~10 becomes ONE `User` owning all their sections ‚Äî never one User per row. Net **200 instructors**. Flag the ~10 email-less ones in the report (can't authenticate via Entra until an email is supplied; their sessions still exist and are connected).

### D8 ‚Äî Preserve instructor‚Üísession wiring
Do not change the `Schedule.faculty` ‚Üí `materialize_sessions` ‚Üí `Session.faculty` path. Every materialized session keeps its direct faculty FK.

### D9 ‚Äî Roomless physical sections ‚Üí TBA placeholder + flag
The 44 physical sections with day/time+instructor but no room: load the schedule anyway pointing at a **"TBA" placeholder room** in the Unassigned building (modality per course mode), and list all 44 in the report for later manual room assignment. Do NOT skip them, do NOT treat the section label as a room. The 14 roomless online sections get the online/virtual treatment (no physical room). 111 no-schedule rows are reported, not invented.

### D10 ‚Äî Reversible clean load
Recommend clearing the term's existing `Schedule`/`Session` rows before a clean load (avoid mixing the `seed_demo` CCIS/IT-30x data with real data), keeping reusable `User`/`Room` rows. Must be reversible (e.g., idempotent/guarded, or a documented undo).
