## Decisions Made
- **Term-scoped building count.** The plan's scale assertion compared a global `Building.objects.count() == 5`, but the dev DB also carries a dormant `seed_demo` "IT Building" (3 rooms, 0 active-term schedules) — exactly what threat T-04.1-08 says stays unmixed with the real term. `tests_e2e.py` scopes the assertion to the buildings the active term actually spans (exactly `ACAD/ADMIN/GYM/ONLINE/UNASSIGNED`). No load defect.
- **e2e assertions as skip-guarded tests + shell helpers.** `manage.py test` spins an empty test DB, so a normal TestCase cannot see the live term. The file skips cleanly there and asserts against the live-loaded `default` DB via `--keepdb` or the shell helpers (the executor ran the equivalent shell assertions live: SCALE OK / SPOTCHECK OK).
