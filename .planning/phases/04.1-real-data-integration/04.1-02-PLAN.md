## Artifacts this plan produces

- `scheduling/management/commands/load_room_master.py` — `load_room_master` command. Reads the 116-tab room-master `.xlsx` via `scheduling.xlsx`, harvests each room tab (tab NAME = authoritative code, C4 = name, O4 = capacity), classifies via `scheduling.importing.classify_room`, and get_or_creates Building/Floor/Room with real name + capacity. Flags: `--file <path>` (default the master xlsx), `--dry-run`. Creates the "Academic Building", "Admin Building", "Gym", "Online (virtual)", and "Unassigned" building rows on demand.
- `scheduling/management/commands/reset_term.py` — `reset_term` command. Flags: `--term-name <name>` (default "2nd Term SY 2025-2026"), `--yes` (required to actually delete). Deletes the term's Session rows then Schedule rows; keeps User/Room/Building. Prints deleted counts. Reversible by re-running the import.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator CLI -> destructive DB delete | reset_term deletes rows; an unguarded run could wipe a term's schedules/sessions unintentionally. |
| room-master `.xlsx` -> DB writes | Untrusted spreadsheet content becomes Building/Floor/Room rows. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04.1-03 | Denial of Service | reset_term command | medium | mitigate | Deletion is gated behind an explicit `--yes` flag; the default run only previews counts. Scope is limited to one named term's Session/Schedule; User/Room/Building are never touched (reversible by re-import). |
| T-04.1-04 | Tampering | reset_term PROTECT handling | low | mitigate | ModalityShiftItem PROTECT is detected and reported (blocked schedule ids skipped), so a partial/ambiguous delete cannot corrupt approved-shift state. |
| T-04.1-SC | Tampering | pip/package installs | high | mitigate | No new dependency (D1); commands reuse stdlib + existing Django + Plan 01 modules. Nothing to install. |
</threat_model>
