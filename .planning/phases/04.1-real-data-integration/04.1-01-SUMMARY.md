## Decisions Made
- **Section-label guard (D9) refined to preserve real rooms.** The plan's literal rule ("a room token equal to Sec is a section label, treat as roomless") would demote `PHY035L`'s `A298` token — but CONTEXT's own room inventory lists `A298` as a real Admin room, and demoting it produced 1041/15 instead of 1042/14. The guard now demotes a `room==Sec` token **only when it resolves to the Unassigned building** (e.g. `C110`), keeping real building rooms like `A298`. This exactly reproduces CONTEXT's 1042/44/14 split and 2021 meetings while still honoring the intent (pure section labels are not rooms). A synthetic test asserts both halves of this distinction.
- **intact = at least one real-room meeting**, with zero-duration `[12:00A-12:00A]` async placeholders excluded as non-meetings. This calibration reproduces every CONTEXT aggregate simultaneously (buckets, 2021 meetings, 168 rooms, 200 instructors, 10 email-less), which is strong evidence it matches the registrar author's manual reconciliation.
- **Typo/unassigned tokens count as rooms but are flagged**, never dropped — `404`/`516` land in `flagged_typo`, `U101`/`U102` in `flagged_unassigned`, so the partition stays exhaustive.

## Deviations from Plan

None that changed scope. One in-scope calibration refinement was applied (documented above under Decisions): the D9 section-label guard was tightened to `room==Sec AND is_unassigned` so a real Admin room (`A298`) sharing its Sec value is not misclassified as roomless. This was necessary to satisfy the plan's own acceptance criteria (exact `1211 = 1042 + 44 + 14 + 111` and `2021` meetings) against the real file, and is explicitly permitted by the plan's grounding note ("tune the bucket logic against the real file … do NOT weaken the 'every row lands in exactly one bucket' guarantee"). The partition remains exhaustive and disjoint.
