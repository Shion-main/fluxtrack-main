# Phase 04.1 Plan 02: Room-Master Loader + Reversible Term-Reset Summary

**Two stdlib-only management commands: `load_room_master` materializes the 114 named master rooms with real capacities (D3) via the D4 prefix map, and `reset_term` gives a `--yes`-guarded, PROTECT-aware, reversible clean-load reset (D10).**

## Accomplishments
- `load_room_master`: reads the 116-tab room-master `.xlsx` through the Plan-01 stdlib reader, harvests each room tab (tab name = authoritative code, C4 = name, O4 = capacity), and get_or_creates Building/Floor/Room with real name + capacity. Tab-skip keys SOLELY on `classify_room(tab_name)` — only `Lecture Summary`/`Lab Summary` skip; U101/P101 and blank-name/blank-capacity rooms still import. Idempotent (refreshes metadata, never rotates tokens); `--dry-run`; per-building report with loud Unassigned flagging.
- Verified against the real master: 114 tabs parsed (ACAD 71, ADMIN 37, GYM 4, UNASSIGNED 2), 2 summary tabs skipped, P101/U101 flagged. All five buildings (incl. Online) created idempotently from the prefix map.
- `reset_term`: resolves the term by `--term-name`; without `--yes` previews Session/Schedule counts and deletes nothing; with `--yes` atomically deletes Sessions then Schedules while preserving User/Room/Building. ModalityShiftItem PROTECT is detected, reported, and skipped rather than crashing. Reversible by re-running load_room_master + import_offerings.

## Decisions Made
- **`is_room_tab` skip predicate:** `classify_room` is total (never rejects) and buckets both the two summary tabs AND the real U101/P101 rooms into `UNASSIGNED`. Keying the skip on `building_code == UNASSIGNED` would wrongly drop U101/P101 (D4/D9 say P/U rooms are real). Instead the predicate keeps a tab when it resolves to a known building OR is a digit-only typo OR carries a floor digit (`floor > 0`), which excludes exactly the two wordy floor-0 summary tabs. This still "keys solely on classify_room(tab_name)" as the plan requires, and the `Room.objects.count() == 114` assertion is the safety net.
- **All five buildings seeded upfront:** the "All Physical" master has no V tabs, so the Online building would never be created from room data. The plan's must-haves require all five buildings to exist, so the command pre-seeds them idempotently from `PREFIX_BUILDING` + `UNASSIGNED_BUILDING` before the room loop.

## Deviations from Plan

None - plan executed exactly as written. The two decisions above are interpretations of the plan's stated intent (they resolve the fact that `classify_room` is total and the physical master lacks V rooms), not scope changes; both are covered by the plan's own acceptance criteria and must-haves.
