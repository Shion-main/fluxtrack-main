# Phase 3 — UI Design Contract

> Visual and interaction contract for the on-duty Checker verification surface. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Prime directive:** this phase adds screens to an app that already ships a Franken UI + htmx look (`base.html`, `templates/faculty/scan.html`, `templates/ifo/live.html`). Do **not** invent a new design system. Every token, class, and pattern below is the one already in the codebase. The floor board is a direct visual sibling of the IFO live "today" view; the scan result screen is a direct sibling of the Faculty `_outcome.html`.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Franken UI via CDN — no `components.json`, no shadcn, no Node build) |
| Preset | not applicable |
| Component library | Franken UI 2.1.2 (`uk-*` component classes over Tailwind utilities), loaded from CDN in `templates/base.html:12-16` |
| Token layer | shadcn-compatible CSS variables via `uk-theme-neutral` on `<html>` (`bg-background`, `text-foreground`, `border-border`, `bg-card`, `text-muted-foreground`, `bg-primary`, `bg-destructive`) |
| Icon library | Lucide, rendered as `<uk-icon icon="name" class="size-4">` (icon.iife.js). Icon names must be valid Lucide names (e.g. `check`, `flag`, `clock`, `arrow-left`, `camera`, `wifi-off`). |
| Font | Franken UI default sans (`uk-font-sm` base on `<html>`); no custom font face is loaded — do not add one |
| Root theme classes | `uk-theme-neutral uk-radii-md uk-shadows-sm uk-font-sm` (inherited from `base.html`; do not override per-page) |
| Rendering | Server-rendered Django templates. Full pages extend `base.html`; interactive fragments are `_name.html` partials swapped by htmx (`hx-swap="innerHTML"`). |

**shadcn gate:** not applicable — stack is Django + htmx (server-rendered), not React/Next.js/Vite. No initialization offered; align to the existing Franken UI system.

## Spacing Scale

Tailwind 4px base (Franken utilities). Declared values (all multiples of 4):

| Token | Value | Tailwind | Usage in this phase |
|-------|-------|----------|---------------------|
| xs | 4px | `gap-1` `p-1` | Icon-to-label gap inside a status pill |
| sm | 8px | `gap-2` `p-2` `mb-2` | Compact stacking inside a card; button icon gap |
| (compact) | 12px | `gap-3` `mb-3` | Card header row gap, list-item internal gap (matches `base.html:27`, `_live_rows.html:2`) |
| md | 16px | `gap-4` `p-4` `px-4` | Default card body padding, page horizontal gutter (`scan.html:7`) |
| lg | 24px | `mb-6` `gap-6` | Section break between coverage header and card grid (`live.html:10`) |
| xl | 32px | `pt-8` | Page top padding (`live.html:4`) |
| 2xl | 48px | `pb-12` | — |
| 3xl | 64px | `pb-16` `pb-24` | Page bottom padding so the offline banner / action bar never overlaps content (`live.html:4`, `scan.html:7`) |

**Exceptions (touch targets — intentional, do not "correct"):**
- Primary tap actions use `h-12` (48px) — Verify, Scan room QR, primary CTA.
- Secondary tap actions and note-submit use `h-11` (44px) — minimum WCAG 2.5.5 target.
- Icon-only buttons use `uk-btn-icon` (min 44×44 hit area) with an `aria-label`.

## Typography

Franken `uk-*` type classes map to these values. Declared: **4 sizes, 2 weights.**

| Role | Size | Weight | Line Height | Class |
|------|------|--------|-------------|-------|
| Meta / label | 14px | 400 regular | 1.5 | `uk-text-small` (+ `uk-text-muted` for secondary) |
| Body | 16px | 400 regular | 1.5 | `text-base` (default body) |
| Section heading | 20px | 600 semibold | 1.25 | `uk-h3` |
| Page heading | 24px | 600 semibold | 1.2 | `uk-h2` |

**Weights:** exactly two — `400` (regular, all body/meta) and `600` (`font-semibold`, headings + emphasized room codes + status labels). Existing `font-medium` (500) room-code cells (`_live_rows.html:14`) consolidate to `font-semibold` in this phase's templates for a single emphasis weight. No other weights.

**Numerals:** the coverage percentage and priority-queue wait times use `text-base`/`uk-h3` at weight 600; do not introduce tabular/mono figures (none loaded).

## Color

Franken `uk-theme-neutral`. The 60/30/10 split is expressed through the semantic token layer; a **separate, explicitly-scoped functional state palette** encodes room verification status (see exception note — this is the heart of CHK-07 and must not be folded into "accent").

| Role | Token / Value | Usage |
|------|---------------|-------|
| Dominant (60%) | `bg-background` (neutral near-white light / `#0a0a0a` dark, `theme-color` `#0a0a0a`) | Page background, board canvas |
| Secondary (30%) | `bg-card` / `bg-muted`, `border-border` | Room cards, coverage panel, list rows, sticky header/banner surfaces |
| Accent (10%) | `bg-primary` / `text-primary` (neutral near-black) | See reserved-for list below |
| Destructive | `bg-destructive` / `text-destructive` / `uk-*-destructive` | Flag actions and flagged/absent status only |

**Accent (`primary`) reserved for — exhaustive list, never "all interactive elements":**
1. The primary CTA button on the scan result screen (`Verify` and `Verified — empty`, `uk-btn-primary`).
2. The coverage progress-bar fill.
3. The "polling" status label on the floor board (`uk-label-primary`, mirrors `live.html:8`).
4. The active/selected state of the floor's own nav affordance.

Everything else (secondary buttons, back button, note-submit, list chrome) uses `uk-btn-secondary` / `uk-btn-ghost` / muted tokens — **not** accent.

### Functional State Palette (documented exception to 60/10 — CHK-07 room-card status)

Color alone is **never** the signal (WCAG 1.4.1): every state carries a Lucide icon **and** a text label in addition to color. Applied only to the room-card left status rail (`border-l-4`), the status pill, and the priority-queue dot — nowhere else.

| State | Meaning | Border / accent | Icon (Lucide) | Pill class / label |
|-------|---------|-----------------|---------------|--------------------|
| Idle | No active session in room | `border-muted` + `text-muted-foreground` | `circle` | `uk-label` · "No session" |
| Active-unverified | Session live, not yet checked | `border-amber-500` + `text-amber-600` | `clock` | `uk-label` amber · "Needs check" |
| Checker-verified | Checker verified present | `border-green-600` + `text-green-600` | `check-check` | `uk-label` green · "Verified" |
| Flagged | Identity mismatch / not present | `border-destructive` + `text-destructive` | `flag` | `uk-label-destructive` · "Flagged" |
| Verified-empty | Confirmed legitimately empty | `border-blue-600` + `text-blue-600` | `door-closed` | `uk-label` blue · "Empty (checked)" |

Absent sessions are **excluded from the board entirely** (CONTEXT/CHK-07) — they never get a card, are not in the coverage denominator, and never appear in the queue. There is no "Absent" state color.

## Copywriting Contract

Voice: plain, second-person, action-first — matches the shipped Faculty copy ("Re-scan this room when your class ends"). No jargon, no emoji in UI text.

| Element | Copy |
|---------|------|
| Primary CTA (scan result, present) | **Verify present** |
| Primary CTA (scan result, empty room) | **Confirm empty** |
| Flag action 1 | **Flag: identity mismatch** (requires note) |
| Flag action 2 | **Flag: not present** (requires note) |
| Online list item CTA | **Open Teams to verify** |
| Note field label / placeholder | "Add a required note" / "Briefly say what you saw (sent to IFO and HR)" |
| Floor board empty state heading | "Nothing to check right now" |
| Floor board empty state body | "No active sessions on your floor. This board refreshes on its own — new sessions appear here as they start." |
| Online list empty state | "No online sessions assigned to you." |
| Priority queue empty state | "You're all caught up. Every active session on this floor is verified." |
| Error — off duty (CHK-01) | Heading "You're not on duty" · body "You don't have an active duty assignment. Ask IFO to post you before verifying." |
| Error — wrong floor (CHK-01) | Heading "Wrong floor" · body "You're on duty for floor {assigned}, but this room is on floor {scanned}. Scan a room on your assigned floor." |
| Error — no Teams link | Heading "No Teams link" · body "This online session has no meeting link. We've flagged IFO to add one." |
| Error — bad / invalid scan | Heading "Invalid code" · body "That is not a valid room QR or six-digit code." (reuse Faculty `_outcome.html` wording) |
| Offline banner | "Offline — {N} scan(s) queued. They'll sync and re-check when you're back online." |
| Offline reconnect — applied | "{N} queued scan(s) applied." |
| Offline reconnect — flagged | "{N} queued scan(s) couldn't apply (the room changed) — sent to IFO to resolve." |
| Loading (board + result) | "Loading…" (reuse `live.html:14`) |

**Destructive / consequential confirmations (CHK-03):**

| Action | Confirmation approach |
|--------|-----------------------|
| Verify present | **One tap, no confirm.** Not destructive; friction-free on the common case. |
| Confirm empty | **One tap, no confirm.** |
| Flag: identity mismatch | **Note-gated, no second confirm.** A required note field appears inline before submit; empty note is rejected server-side (render error partial, never 500). On submit, fires immediately to IFO + HR. Copy under the field: "Flags reach IFO and HR permanently and can't be appealed." |
| Flag: not present | Same note-gated pattern as above. |

There is **no** "Confirm absent" action anywhere — Absent is the sweep's job (CHK-06 removed; `CONFIRMED_ABSENT` retired). Do not add such a button.

## Component Specs

All components are Django templates. Full page = extends `base.html`; fragment = `_name.html` swapped via htmx.

### 1. Floor board shell — `templates/checker/floor.html`
- Extends `base.html`. Container `uk-container max-w-4xl pt-8 pb-16` (sibling of `live.html`).
- Header row (`flex items-center gap-2 mb-1`): back `uk-btn-ghost uk-btn-icon` (`arrow-left`, `aria-label="Back"`), `<h1 class="uk-h2">Floor {n}</h1>`, `<span class="uk-label uk-label-primary">polling</span>`.
- Sub-line `uk-text-muted mb-6`: "Your assigned floor, refreshing automatically."
- **Coverage panel** (see #2) then the **polled rows** container:
  `<div id="floor-rows" hx-get="/checker/floor/rows" hx-trigger="load, every {{ poll_ms }}ms" hx-swap="innerHTML"><p class="uk-text-muted uk-text-small">Loading…</p></div>`
  `poll_ms = settings.FLUXTRACK_POLICY["poll_interval_seconds"] * 1000` — **never hardcode** (Convention rule #3).

### 2. Coverage indicator — inside `templates/checker/_floor_rows.html`
- Panel `uk-card uk-card-body` with a header row (`flex items-center justify-between mb-3`).
- Left: `<span class="uk-h3">{{ verified }}/{{ total }} verified</span>`. Right: `<span class="uk-label uk-label-secondary">updated {{ now|time:"g:i:s A" }}</span>` (mirrors `_live_rows.html:4`).
- Progress bar: `<progress class="uk-progress" value="{{ verified }}" max="{{ total }}">` OR a `bg-muted` track with a `bg-primary` fill div at `width:{{coverage}}%`. Include `aria-label="Coverage {{ coverage }} percent"`.
- Denominator excludes Absent (Pitfall 5) — same queryset feeds cards, queue, and this count.

### 3. Priority queue — top of `_floor_rows.html`
- Heading `uk-text-small font-semibold mb-2`: "Check next (oldest first)".
- Ordered list of active-unverified sessions, `order_by("scheduled_start")` (longest-waiting on top).
- Each row: a colored status dot (state palette), room code (`font-semibold`), course-section, and a wait badge `uk-label` reading the elapsed time (e.g. "22m waiting"). Row is a link/tap target to that room's scan/detail; min height 44px.
- Empty → queue empty-state copy.

### 4. Room card — repeated in `_floor_rows.html`
- `uk-card uk-card-body` with `border-l-4 {state border}` status rail.
- Top row (`flex items-center gap-2`): status icon + `<span class="uk-label ...">{state label}</span>` + room code `font-semibold ml-auto`.
- Body: course-section + scheduled faculty last name (`uk-text-small`).
- Whole card is a tap target (min 44px height) linking into the scan result for that room. Grid is mobile-first single column, `sm:grid-cols-2 lg:grid-cols-3` on wider screens.
- F2F/Blended rooms only; online never appears here.

### 5. Scan result screen — `templates/checker/scan.html` + `_outcome.html`
- Page mirrors `faculty/scan.html`: `max-w-md mx-auto px-4 pt-4 pb-24`, camera button `uk-btn-primary w-full h-12` ("Scan room QR", `camera` icon), `#qr-reader`, an "or" divider, and a six-digit manual `uk-input h-12 text-center text-xl tracking-widest`. Reuse the `html5-qrcode` + `postPayload` script pattern verbatim.
- Outcome swaps into `#outcome` (`_outcome.html`), a flat `{% if/elif %}` chain over resolver outcomes, each with a `data-outcome="..."` hook (mirrors Faculty).
- **active-unverified outcome** renders: faculty **profile photo** (see #6) + faculty name + course/section/room, then the **action bar** (#7).
- **off-duty / wrong-floor** outcomes render the refusal alert (see #8) — no action bar.
- **online** outcome does not render a room card; it routes to the Teams flow (online sessions are scanned only from the online list, not the floor board — but if a room QR resolves to an online context, show the "verify online via Teams" alert).

### 6. Faculty identity photo (CHK-02)
- `accounts.User.profile_photo` (`ImageField`, exists). Render `<img class="size-16 rounded-full object-cover border border-border">` (64px).
- **Fallback** when no photo: initials avatar — `<div class="size-16 rounded-full bg-muted text-muted-foreground flex items-center justify-center uk-h3">{{ initials }}</div>`. Confirm `MEDIA_URL`/`MEDIA_ROOT` serve in DEBUG so `.url` renders (RESEARCH RESOLVED-03-02).
- `alt="Photo of {{ faculty.get_full_name }}"`.

### 7. Action bar (CHK-03) — inside the active-unverified `_outcome.html` branch
- Vertical stack, full-width, mobile-first.
- **Row 1 (one-tap):** `Verify present` — `uk-btn uk-btn-primary w-full h-12`, posts immediately (`hx-post="/checker/action" hx-target="#outcome"`, hidden `action=verified` + signed token). For an empty room this button reads `Confirm empty` (`action=verified_empty`).
- **Row 2 (flags):** two `uk-btn uk-btn-secondary w-full h-11` buttons: "Flag: identity mismatch" and "Flag: not present". Tapping one reveals an inline note form (does not navigate) — the button `hx-get`s a note-form partial into a slot, OR toggles a hidden `<div>` containing:
  - `<textarea name="note" class="uk-input" required maxlength="255" placeholder="…">`, helper line "Flags reach IFO and HR permanently and can't be appealed.", and a `uk-btn uk-btn-destructive w-full h-11` submit ("Send flag").
- Server rejects a flag with empty note by re-rendering the form partial with `uk-form-danger` + inline error (Pitfall 4) — never a 500.
- No second confirm step for any action (CONTEXT).

### 8. Refusal message (CHK-01 gating)
- `uk-alert uk-alert-secondary` (not destructive — it's a guard, not an error) with `uk-alert-title` + `uk-alert-description uk-text-small`, `data-outcome="off-duty"` / `data-outcome="wrong-floor"`.
- Copy from the Copywriting Contract; wrong-floor interpolates `{assigned}` and `{scanned}` floor numbers so the reason is specific (CONTEXT: "clear, specific reason").
- No action bar rendered.

### 9. Online-to-verify list — `templates/checker/_online_list.html`
- Separate surface from the floor board (own route/section), only shown to online-duty Checkers, listing sessions where `Session.online_checker == request.user`.
- Each item `uk-card uk-card-body flex items-center gap-3`: course-section + faculty name + scheduled time on the left; a `uk-btn uk-btn-primary h-11` "Open Teams to verify" on the right that opens `session.teams_link` (`target="_blank" rel="noopener"`).
- After the Checker returns from Teams, they record Verify present / Flag: not present via the same action endpoint (online branch). Missing `teams_link` → render the "No Teams link" alert instead of the button.
- Empty → online empty-state copy.

### 10. Offline banner (CHK-08) — global partial, `templates/checker/_offline_banner.html`
- Sticky bar under the header, `bg-muted border-b border-border`, hidden by default, shown by the IndexedDB queue JS when `!navigator.onLine` or queue length > 0.
- Content: `wifi-off` icon + "Offline — {N} scan(s) queued…". On reconnect, swap to the applied/flagged summary copy; flagged summary uses `text-destructive`.
- Never blocks interaction; it is a status strip, not a modal.

## States (every surface must define these)

| Surface | Loading | Empty | Error | Success |
|---------|---------|-------|-------|---------|
| Floor board rows | "Loading…" muted line | "Nothing to check right now" + body | polled fetch fails → keep last render, show muted "reconnecting…" (htmx `hx-on::response-error`) | new/verified cards appear on next poll tick |
| Scan result | inline spinner via htmx `hx-indicator` | "No session due in {room} right now." (verified-empty path) | invalid code / bad payload alert | green "Verified" alert, `data-outcome="verified"` |
| Action (flag) | button `uk-disabled` while posting | — | note-required inline error | flag alert + "sent to IFO and HR" |
| Online list | "Loading…" | "No online sessions assigned to you." | no-link alert | verified item drops from list on next load |
| Offline queue | — | banner hidden | flagged-on-replay summary (`text-destructive`) | applied summary |

**Polling/loading detail:** the board polls on the policy interval (default 8s). Use htmx `hx-indicator` for the manual scan post; do **not** show a full-screen spinner on the auto-poll (it must feel ambient, like the IFO live view). Preserve scroll position across swaps.

## Accessibility (WCAG 2.1 AA)

- **1.4.1 Use of Color:** every room state = color + Lucide icon + text label. No state is distinguishable by hue alone.
- **1.4.3 Contrast:** body/label text on `background`/`card` meets 4.5:1; the amber state uses `text-amber-600` (not `amber-400`) on light surfaces to hold contrast; verify green/blue/amber label text ≥ 4.5:1 against the card. Status pills carry text, so the pill fill need only meet 3:1 (non-text UI, 1.4.11).
- **2.5.5 Target Size:** all tap targets ≥ 44×44 (`h-11`/`h-12`, `uk-btn-icon`).
- **1.1.1 Non-text Content:** faculty photo has a descriptive `alt`; icon-only buttons have `aria-label` (matches `scan.html:9`).
- **4.1.2 / status:** the offline banner and poll-updated coverage use `aria-live="polite"`; the flag-error uses `role="alert"`.
- **2.4.7 Focus Visible:** rely on Franken UI default focus rings; do not remove outlines. Ensure the whole-card link and queue rows are keyboard-reachable (`<a>`/`<button>`, not `onclick` divs).
- **Forms:** note `<textarea>` has an associated `<label>`; `required`/`maxlength` are UX only — the server is the gate.
