<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| checker browser → resolve/action endpoints | Untrusted payload (room token/code, action, note, identity_match) crosses here |
| checker → faculty PII (photo) | On-duty checker views a scheduled faculty member's photo for identity matching |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-03 | Elevation of Privilege | off-duty / wrong-floor scan | high | mitigate | `checker_required` + server-side `_active_floor_ids` gating via the pure core; the client never supplies its own floor — the scanned room's floor is derived server-side and checked against active assignments |
| T-03-04 | Repudiation | flag with no reason reaching HR permanently | high | mitigate | Server-side rejects FLAG_* with empty note (renders error partial, writes nothing); every accepted action writes an AuditLog row (actor + event_type) |
| T-03-05 | Spoofing | forged action POST (verify someone else's session) | medium | mitigate | `checker_required` + `@require_http_methods(["POST"])` + CSRF (global hx-headers); resolution/session fetched server-side, not trusted from the client |
| T-03-06 | Information disclosure | faculty photo exposed to a non-on-duty user | medium | mitigate | Photo only rendered inside an `active-unverified` outcome, which is only reachable when `_active_floor_ids` includes the room's floor |
| T-03-07 | Denial of Service | manual-code brute force | low | mitigate | Reuse the per-user-per-minute cache rate limit from web/scan.py; audit `checker.rate_limited` |
</threat_model>
