<objective>
Deliver the IFO-facing side of IFO-06: a non-admin UI where IFO assigns Checkers and Guards to floors (shift or standing) and grants online duty (scope=ONLINE), plus the round-robin apply layer that pre-assigns each online session to exactly one online-duty Checker (or leaves it unassigned with an IFO flag when no online-duty Checker exists).

Purpose: On-duty powers come only from an active Assignment (CHK-01 gating in 03-02 already reads them). This plan is how those assignments get created by a human, and how online sessions get their one accountable owner before 03-05's online verification runs. Guard assignments are created here too (IFO-06), but Guard-facing screens stay in Phase 7.
Output: IFO assignment views + templates, `verification/services.py` (assign apply), the `assign_online` management command, and the assignment/distribution tests.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| IFO browser → assignment-create endpoint | Untrusted assignment params (user, role, scope, floors, times) cross here |
| assign apply → Session ownership | Writes who is accountable for verifying each online session |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-08 | Elevation of Privilege | non-IFO creating duty grants | high | mitigate | `ifo_required` on both assignment views + `@require_http_methods(["POST"])` + CSRF; assignment creation is the on-duty-power root, so it must be IFO-only |
| T-03-09 | Tampering | forged scope/floor to self-grant online duty | medium | mitigate | Server builds the Assignment from validated fields; scope constrained to AssignmentScope choices; FLOOR requires real floor pks; every create writes an AuditLog |
| T-03-10 | Repudiation | silent unassigned online sessions | medium | mitigate | Empty online roster leaves online_checker NULL and fires notify(IFO online_unassigned) rather than guessing — a human is told, and each assignment is audited |
| T-03-11 | Denial of Service | HY010 on batch online assignment (MSSQL) | medium | mitigate | Candidate sessions materialized with list() before the write loop (no `.iterator()`), mirroring scheduling/jobs.py |
| T-03-SC | Tampering | package installs | low | accept | No new packages this phase (03-RESEARCH § Package Legitimacy Audit) |
</threat_model>
