## Decisions Made
- **Merge key is exact-instant + faculty + (room OR course).** `scheduled_start` is compared as the full aware DateTime; a 1-minute offset disqualifies. Matches D-01 and the criterion #5 sibling set exactly.
- **Status guard lives on the `.update()` filter**, not a Python loop — the single filtered `UPDATE (... WHERE status=SCHEDULED)` gives idempotency and the D-05 no-clobber guarantee for free while staying HY010-safe.
- **Anchor is never re-stamped MERGED** — MERGED identifies the auto-filled sibling, keeping reporting able to tell the scanned row from the propagated ones.
- **Online merged pair uses a distinct start time** from the F2F group in the fixture, so `propagate_merged_absent(online_anchor)` and `propagate_merged_present(anchor)` operate on cleanly independent candidate sets.
