## Identity → Seeded-User Linking

| Option | Description | Selected |
|--------|-------------|----------|
| Email/UPN match, link on first login | social-auth matches by email/UPN, writes azure_oid on first login, then matches by azure_oid; no match = refused (no auto-create) | ✓ |
| Pre-set azure_oid manually | Paste each person's Entra object id onto their User row ahead of time; match azure_oid only | |
| Auto-provision from Entra | Any tenant account creates a User on first login | |

**User's choice:** Email/UPN match, link on first login.
**Notes:** Enforces "unprovisioned = refused". azure_oid is the durable key; email is the first-login bridge. Safe because AzureADTenantOAuth2 is single-tenant (email tenant-verified).
