<objective>
Own the app-level identity policy: refuse any authenticated tenant identity that has no
pre-provisioned, active seeded User (D-06/AUTH-03/AUTH-05), persist the durable Entra
object id on first successful link (D-05), and provide the one-off `link_entra` command
that binds a seeded slot to a real MMCM UPN so email-match can link (D-07).

Purpose: The library supplies the OAuth/JWKS/PKCE/linking primitives; this plan writes
the ~2 custom pipeline steps + 1 management command that encode FluxTrack's
"provisioned-users-only" rule and the proof's linking mechanic. Login success/refusal
are security events, so each writes an AuditLog row (Convention #2 — planner's discretion
exercised in favor of the project's hard audit convention).

Output: `accounts/pipeline.py` (deny_unprovisioned + write_azure_oid) and
`accounts/management/commands/link_entra.py`.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03.1-04 | Elevation of privilege | SOCIAL_AUTH_PIPELINE provisioning | critical | mitigate | create_user removed (Plan 01) + deny_unprovisioned raises AuthForbidden when no seeded User matches (D-06/AUTH-03) |
| T-03.1-05 | Elevation of privilege | reactivation/inactive bypass | high | mitigate | deny_unprovisioned also refuses `not is_active` (AUTH-05) |
| T-03.1-06 | Spoofing | account takeover via email match | high | mitigate | Single-tenant backend enforces `tid` (org-verified email, D-05); associate_by_email raises on >1 match; write_azure_oid persists the durable oid so later logins match by oid |
| T-03.1-07 | Repudiation | missing audit trail of auth security events | medium | mitigate | deny_unprovisioned emits auth.entra_refused; write_azure_oid emits auth.entra_login (Convention #2) |
| T-03.1-12 | Tampering | link_entra email-clash creating a duplicate match | medium | mitigate | Cross-user case-insensitive clash check raises CommandError before writing (prevents associate_by_email >1-match failure) |
