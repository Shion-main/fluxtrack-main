# Phase 03.1 Plan 03: Dev-login / Microsoft SSO Coexistence Summary

Made the two login paths coexist on localhost: the DEBUG dev-login POST now names
ModelBackend explicitly so it survives the second AUTHENTICATION_BACKENDS entry added
in Plan 01, and the real "Sign in with Microsoft" button is always visible on `/login`
(both DEBUG and non-DEBUG) wired to `social:begin`, with refusal messages surfaced.

## What Was Built

### Task 1 — Name ModelBackend in the dev-login stub (`web/views.py`)
- Changed `login(request, user)` to
  `login(request, user, backend="django.contrib.auth.backends.ModelBackend")`.
  With two configured backends Django's `login()` cannot infer which authenticated the
  user and raises `ValueError`; naming ModelBackend fixes the dev-login 500 (Pitfall 2).
- Refreshed the `login_view` docstring: Microsoft Entra SSO is now described as the real
  sign-in path and the username dev-login as a DEBUG-only convenience (removed the stale
  "not yet implemented" / "auth cutover phase" wording — Pitfall 8).
- `logout_view` deliberately left unchanged (D-11 — local session flush only).
- Commit: `9c03368`

### Task 2 — Always-visible Microsoft button + refreshed env (`templates/web/login.html`, `.env.example`)
- Moved the "Sign in with Microsoft" link out of the `{% else %}` branch so it renders in
  both DEBUG and non-DEBUG (D-08), and pointed its href at
  `{% url 'social:begin' 'azuread-tenant-oauth2' %}` (was `#`). Franken UI classes
  (`uk-btn uk-btn-primary w-full h-11`) preserved.
- Added a Django-messages block (`uk-alert-danger`) so a refused Entra login (surfaced by
  `SocialAuthExceptionMiddleware`) is visible rather than a blank page.
- Kept the per-user dev-login forms inside the `{% if dev_mode %}` block; replaced the
  stale "Entra ID SSO wired in Phase 2" copy with phase-agnostic "Dev login (DEBUG only)".
- `.env.example`: replaced the commented `ENTRA_TENANT_ID/CLIENT_ID/CLIENT_SECRET` block
  with `SOCIAL_AUTH_AZUREAD_TENANT_OAUTH2_KEY` (client id), `_SECRET` (client secret),
  `_TENANT_ID` (directory id) per D-04, and dropped the "Phase 2" label (Pitfall 8).
- Commit: `e3b2cde`
