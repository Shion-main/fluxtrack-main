# Phase 03.1 Plan 02: App-Level Identity Policy (pipeline steps + link_entra) Summary

Wrote the ~all of FluxTrack's app-owned Entra auth logic: two custom
`SOCIAL_AUTH_PIPELINE` steps that enforce the "provisioned-users-only" rule and
persist the durable Entra object id, plus the one-off `link_entra` command that
binds a seeded User slot to a real MMCM UPN so first-login email-match can link.
Both auth security events emit an `AuditLog` row. The forward-reference the Plan 01
settings block left dangling (`accounts.pipeline.deny_unprovisioned` /
`write_azure_oid`) now resolves; `seed_demo.py` is untouched.

## What Was Built

**Task 1 — Custom pipeline steps + security audit** (`2636fec`, `accounts/pipeline.py`)
- `deny_unprovisioned(strategy, backend, details, user=None, *args, **kwargs)` —
  runs AFTER `associate_by_email` with `create_user` removed (both wired in Plan 01),
  so `user is None` means the tenant-verified identity matched no seeded User. Refuses
  `user is None or not user.is_active` (the `is_active` branch covers AUTH-05
  reactivation bypass) by writing an `auth.entra_refused` AuditLog row
  (`actor=None`, attempted email in `payload`) then raising `AuthForbidden(backend)`
  — which `SocialAuthExceptionMiddleware` (Plan 01) routes to
  `SOCIAL_AUTH_LOGIN_ERROR_URL=/login`. Returns `None` for a provisioned, active user
  (pipeline continues). (D-06/AUTH-03/AUTH-05)
- `write_azure_oid(strategy, backend, details, response, user=None, *args, **kwargs)` —
  no-ops on `user is None`; reads `oid = response.get("oid")` explicitly (the social
  uid is the pairwise `sub` claim, NOT the durable object id); writes
  `user.azure_oid` via `save(update_fields=["azure_oid"])` only when the oid is
  present and changed (idempotent); on a non-None-user call writes an
  `auth.entra_login` AuditLog row (`actor=user`, `target_type="user"`,
  `target_id=str(user.pk)`). `oid` may be absent (Assumption A1) — the write is then
  a safe no-op and login still succeeds via the UserSocialAuth uid. (D-05)
- Module docstring cites D-05/D-06 + AUTH-03/05 (module-docstring-cites-requirement
  convention); the AuditLog calls mirror the house
  `AuditLog.objects.create(actor=..., event_type=..., payload=...)` shape from
  `web/scan.py` without importing it.

**Task 2 — link_entra management command** (`b2fa47c`, `accounts/management/commands/link_entra.py`)
- `BaseCommand` with two positional args (`username`, `upn`) and D-07 help text.
- `handle()` resolves `get_user_model()`, validates the UPN with
  `django.core.validators.validate_email` (`CommandError` on failure), fetches the
  user by username (`CommandError` if absent), rejects a cross-user email clash via a
  case-insensitive `filter(email__iexact=...).exclude(pk=user.pk)` check
  (`CommandError` — prevents an `associate_by_email` >1-match failure, threat
  T-03.1-12), prints a `No change -> ...` SUCCESS line and returns 0 when the email
  already equals the UPN (idempotent), otherwise sets `user.email` +
  `save(update_fields=["email"])` and prints `Linked <username> -> <upn>`.
- ASCII-only output (`->` arrows, no Unicode/emoji) per Convention #4. Module
  docstring cites D-07 and notes the command does not mutate `seed_demo.py` (the
  committed demo seed stays synthetic).
