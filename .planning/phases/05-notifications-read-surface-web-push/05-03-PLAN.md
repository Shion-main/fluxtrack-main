<objective>
Deliver web push fault-isolated (criterion #4): a DB-backed outbox polled by the existing single
BlockingScheduler process. The scan/approval/job that emits notify() only writes Notification rows;
the scheduler's push_outbox job later reads unpushed key-event rows, sends via pywebpush, prunes
dead endpoints, and stamps pushed_at — so a hung or dead push endpoint is structurally incapable of
touching the triggering request (D-09).

Purpose: Satisfy the "failed push never breaks the trigger" criterion by construction, not by
timeout luck; unify web-originated (scan) and scheduler-originated (weekly report) push through one
sender; keep all 410/404 pruning inside the scheduler process per the one-process rule.
Output: `ops/push.py` sender + pruning, the registered `push_outbox` job, and a fully mocked test
suite (Nyquist Wave 0: ops/tests_push.py).
</objective>

  <behavior>
    - _send_one(sub, payload) returns True on delivery, False ONLY on WebPushException with status
      404 or 410 (dead endpoint), and never raises for a failed send.
    - A transient failure (429/5xx/timeout/network) returns True (do NOT prune) — the row is treated
      as handled this pass so it is not retried forever.
    - send_push_outbox() selects Notification rows where type in PUSH_TYPES, pushed_at IS NULL, and
      created_at within a recent window; returns the count of rows it processed to send.
    - If the recipient has muted the row's category (muted_types), no webpush is attempted but
      pushed_at is stamped (D-05 suppresses push).
    - After processing a row, pushed_at is stamped exactly once (so it is never re-sent).
    - When VAPID is unconfigured (empty VAPID_PRIVATE_KEY_PATH), send_push_outbox short-circuits and
      sends nothing (push disabled) without error.
  </behavior>
  <action>
Create `ops/push.py` with a module docstring citing NOTIF-02 / criterion #4 / D-08 / D-09. Import
`webpush` and `WebPushException` from pywebpush, `settings`, `timezone`, `timedelta`, the ORM models
`Notification` and `PushSubscription` from ops.models, and `PUSH_TYPES` + `muted_types` from
ops.notifications (the SAME filter the list uses — D-06). Do NOT re-implement any VAPID crypto; the
library owns it (do-not-hand-roll).

Implement `_send_one(sub, payload)`: call `webpush(subscription_info={"endpoint": sub.endpoint,
"keys": sub.keys}, data=json.dumps(payload), vapid_private_key=settings.VAPID_PRIVATE_KEY_PATH,
vapid_claims={"sub": settings.VAPID_SUB}, ttl=600, timeout=10)` inside try/except. `timeout=10` is a
hard cap so a hung endpoint can never stall the job (threat T-05-05). On WebPushException, read
`exc.response.status_code` defensively (response may be None); return False for 404/410 (prune, D-09)
and True for any other status (transient — leave the subscription, treat as handled this pass).

Implement `send_push_outbox()`: if `not settings.VAPID_PRIVATE_KEY_PATH`, return 0 (push disabled).

`Notification.objects.filter(type__in=PUSH_TYPES, pushed_at__isnull=True, created_at__gte=window)

continue (D-05, no send). Otherwise build `payload = {"title": n.title, "body": n.body,

`_send_one`; if it returns False, `sub.delete()` (prune, D-09). After the loop stamp
`n.pushed_at = timezone.now()` via `save(update_fields=["pushed_at"])` and count it. Return the sent

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| scheduler -> push vendor endpoint | Outbound HTTPS POST to browser-vendor endpoints (FCM/Mozilla/Apple) stored from real subscriptions |
| triggering request -> push path | The web request that emits notify() must never enter the push send path |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-05 | Denial of Service | _send_one webpush call | high | mitigate | Hard `timeout=10` per send + broad except that never raises; runs in the scheduler, never a web worker, so a hung endpoint cannot stall a request (criterion #4). |
| T-05-07 | Denial of Service | send_push_outbox pass | high | mitigate | Wrapped by run_job (records failed, never re-raises); windowed scan (created_at >= now-15m) bounds work; max_instances=1 + coalesce prevent pile-up. |
| T-05-08 | Tampering | dead-endpoint pruning | medium | mitigate | Prune ONLY on 404/410; transient 5xx/timeout kept, so a flaky vendor does not silently drop live subscriptions. |
| T-05-13 | Server-Side Request Forgery | outbound POST targets | low | accept | Endpoints come only from validated PushSubscription rows (endpoint validated at subscribe time in 05-05); the sender never follows arbitrary user input. |
</threat_model>
