**Required structure — do not improvise this.** Today `_cards.html` wraps ALL
four cards inside a single `{% if summary.1 %}...{% else %}<div class="grid">`,

card into that `{% else %}` branch would make a `dept_summary` failure delete the

looking correct on a healthy page.

1. **Lift the `grid` div ABOVE the summary guard**, so the row container exists
   regardless of which card errored.
2. **Inside that grid, guard the four summary cards as one unit** — on failure

3. **Inside the same grid, guard the occupancy card independently** in its own
   cell.

the occupancy cell still shows its figure, and an occupancy failure renders an
error in one cell while the four attendance cards still show theirs. Two error
owners, one row, neither able to take the other down.

Handle the in-flight count: when it is non-zero, add a short muted line stating
