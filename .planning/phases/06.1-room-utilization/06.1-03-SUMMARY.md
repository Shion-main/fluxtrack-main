## The Paginate Bug Was Real, and Is Now Pinned

`paginate(request, rows[0])` handed `Paginator` the `None` that `safe_card` returns
when `faculty_attendance` raises. Reproduced before the fix:

```
RuntimeError: aggregate exploded
TypeError: object of type 'NoneType' has no len()
```

An aggregate failure 500'd the page, defeating the exact isolation `safe_card`
exists to provide. Fixed as `rows[0] or []` with a comment naming the guard and
citing its test. **Reverting the guard turns exactly one test red** —
`DashboardCardIsolationTests.test_faculty_attendance_failure_does_not_500_via_paginate`
— and nothing else, so it binds to the real bug and only to it.

## The Template Structure, and Proof the Nesting Trap Is Caught

Before, the `grid` div lived INSIDE the summary guard's `{% else %}` branch, so
dropping the occupancy card in there would have made a `dept_summary` failure
delete the occupancy card too — invisible on a healthy page. Shipped structure:

1. `<div class="grid gap-4 sm:grid-cols-2 md:grid-cols-3 lg:grid-cols-5">` — the
   row container, ABOVE every guard.
2. Inside it, the four summary cells guarded as one unit on `summary.1`.
3. Inside the same grid, the occupancy cell guarded independently on `occupancy.1`.

Verified by mutation: nesting the occupancy block inside the summary guard turns
**exactly one test red** —
`test_dept_summary_failure_leaves_the_occupancy_card_standing`, which asserts the
session-hours caption STRING survives, not merely that the page rendered. Both
mutations were reverted clean (`git diff --stat` empty).

## Deviations from Plan

### Judgement Calls Made

**The utilization pill is `pill--info` (neutral), not the ok/warn/bad ladder.**
The plan said to use "the existing `pill` vocabulary from line 66", which is the
attendance ladder. Applied literally, 26% — the campus's genuine baseline, because
the 60/20/20 modality split leaves physical capacity idle by design — renders as a
red `pill--bad`. A standing red on the normal state is a false alarm that teaches a
reader to distrust the number, which is the opposite of this phase's purpose. There
is no decision in `06.1-CONTEXT.md` establishing a utilization target, so inventing
thresholds would have been inventing policy. `pill--info` is already IFO vocabulary
(`templates/ifo/weekly_reports.html:47`) and the percentage is printed inside the
pill, so colour is still not the only signal.

**The occupancy headline is `used_hours` alone, with the denominator in the
caption.** The plan asked for "the used hours as the primary figure with booked
hours as the muted denominator". Built that way first and looked at it: at
`lg:grid-cols-5` the string "2707.9 / 3180.0" wraps mid-figure into "2707.9 /" over
"3180.0", reading as two unrelated numbers. The plan itself predicted this ("tight
for the two-part used / booked value"). Restructured to a non-wrapping headline
plus "session-hours used of 3180.0 booked", which keeps both figures and the unit
in view. Found by rendering the page, not by reading the template.

**Task 3's tests were written FIRST, not last.** Task 1 carries `tdd="true"` and its
`<behavior>` block is entirely about dashboard responses, so writing Task 3's file
last would have inverted the RED/GREEN order the task demands. The whole file was
committed failing as the RED gate, then the view and template turned it green.

**Two `DashboardRangeTests` beyond the plan's spec** (Rule 2):
`test_a_narrower_range_yields_a_smaller_occupancy_figure` (the plan's human check
"confirm the figure changes when the date range changes", made automatic) and
`test_the_active_term_reaches_the_aggregate_so_the_ladder_is_derived` (proves the
term lookup actually feeds the ladder, not just that a term was passed).

## Plan Assumptions That Turned Out Wrong

**`AcademicTerm` was already imported in `web/ifo.py`.** The plan said to add it "if
it is not already imported in this module". It is, at line 32. Only
`room_utilization` needed adding to the `scheduling.reporting` import.

**No Wave 1 API drift affected this plan.** Unlike plan 02, plan 03 consumes only
`room_utilization` and `RoomUtilization`, both of which survived the D-10 amendment
with their signatures intact. The new `timetabled_cells` field is available on the
dataclass but is not rendered — `physical_rooms`, `blocks_per_day` and
`teaching_days` are denominator-shape diagnostics rather than stakeholder-facing
figures, and the plan's card spec does not call for them.
