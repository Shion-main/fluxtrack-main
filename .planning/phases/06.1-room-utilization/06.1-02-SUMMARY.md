# Phase 06.1 Plan 02: Room Fixture + DB-Backed Metric Tests Summary

A room-shaped fixture with real `actual_start`/`actual_end` variety, 27 new
DB-backed tests pinning every decision-level claim of the T1 metric, and a
falsifiable shell recipe for judging the live campus figure by hand.

## What Was Built

**Task 1 — `make_room_utilization_fixture`** (`b29bd87`). A new function beside
`make_reporting_fixture` in `scheduling/test_support.py`. Two buildings x two
floors x two rooms = **8 physical rooms**, plus a third building holding one
V-prefixed virtual room. A derived **3-rung ladder** (07:00 / 08:15 / 09:30) with
**two distinct durations** (75 / 75 / 90 min), timetabled on Monday, Wednesday and
**Saturday** and never on Sunday. Eleven named sessions, one per used-hours shape.
`activate=False` on any second call keeps exactly one `is_active` term.

**Task 2 — DB-backed decision tests** (`4489674`). `LadderDerivationTests`,
`UsedHoursPolicyTests`, `VirtualRoomExclusionTests`, `ScopeTests` and
`RoomCardIsolationTests`, on a shared `RoomFixtureTestCase` base carrying the
fixture's documented minute totals.

**Task 3 — the seed sanity recipe** (`18f0d15`). A copy-pasteable
`manage.py shell -c` one-liner at the foot of `tests_reporting_rooms.py`, plus the
four checks that make its output falsifiable. Deliberately not assertions.

## Deviations from Plan

### Judgement Calls Made

**`DenominatorTests` was NOT written — it already exists.** The plan asked for a
class asserting `available_hours` against an independent recomputation and a
zero-schedule term degrading to 0. The Wave 1 D-10 amendment already shipped
exactly that (9 DB-backed tests with `_expected_available_seconds` rebuilding the
denominator in plain Python). Duplicating it would have added a second, weaker
copy. Its coverage is instead extended sideways: `LadderDerivationTests` pins the
fixture's own shape and `ScopeTests` pins the `as_of` clamp on the numerator.

**Delta assertions are written as "the new total", not "the difference".**
`_hours` quantizes to one decimal place with ROUND_HALF_UP, so
`_hours(a) - _hours(b) != _hours(a - b)` in general (630 min -> 10.5,
555 min -> 9.3, difference 1.2, but 75 min -> 1.3). Every per-row test therefore
deletes one session and asserts the resulting total equals
`_hours((DOCUMENTED_M - row_m) * 60)`, which is exact.

**A `RoomFixtureTestCase` base class was added** beyond the plan's named classes,
so the fixture setup and the documented totals appear once rather than in five
places. It declares no `test_` methods and so contributes no tests itself.

**`test_only_one_session_carries_the_ended_early_flag` was added** (Rule 2). It
pins the flag/timestamp DISAGREEMENT itself — `early_end_sessions == 1` while two
sessions genuinely ended early and both remainders are in the waste figure — so a
future reader cannot argue the unflagged test is redundant without this also
going red.

**`test_a_range_carrying_no_timetabled_day_is_zero_not_a_crash` was added**
(Rule 2), covering the Sunday-only range: `teaching_days` 0, `available_hours`
0.0, `utilization_pct` 0, no exception.

## Plan Assumptions That Turned Out Wrong

**`teaching_weekdays` no longer exists.** The plan's `LadderDerivationTests` spec
instructed asserting `teaching_weekdays(term)` contains `DayOfWeek.SAT` and not
`DayOfWeek.SUN`. That helper was REMOVED by the D-10 amendment after this plan was
written. The claim is preserved verbatim in
`test_saturday_is_picked_up_and_sunday_is_absent_from_the_data`, using
`{c.day for c in timetabled_cells(term)}` — the day set is now recovered from the
cell set, which is the actual unit of capacity.

**`Room.manual_code` cannot be prefix-namespaced.** The plan required two calls
with different prefixes to coexist. `make_reporting_fixture`'s existing idiom is
`f"{prefix[:1].upper()}{n:05d}"[:6]`, which depends only on the prefix's FIRST
letter — so `"rutil"` and `"rutil2"` both yield `R00001` and collide on a UNIQUE
CS_AS column. The room fixture uses a process-wide `itertools.count` instead. The
existing fixtures were left alone; they are only ever called once per test.

**`Room.code` is CI-collated AND unique, so the lowercase-v test cannot reuse the
V-room's code.** `"vrutil1"` would collide with `"Vrutil1"` on the unique index
rather than testing the exclusion. `test_a_lowercase_v_room_is_excluded_too`
creates a distinct `"vlowercase1"` instead.
