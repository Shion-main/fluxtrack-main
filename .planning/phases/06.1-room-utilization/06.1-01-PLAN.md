<metric_contract>
Read this before writing code. Every number below is fixed here and referenced by
plans 02-07; do not redefine any of it downstream.

**Scope of every room aggregate in this phase.** Start from `_scoped_sessions`
(inherits `date__range` on the LOCAL `Session.date` and
`schedule__status=ScheduleStatus.ACTIVE`, plus the `as_of` clamp), then apply the
physical-room exclusion. Never filter on the UTC `scheduled_start`.

**BOOKED seconds** = sum over scoped sessions of
`scheduled_end - scheduled_start`.

**USED seconds** = sum over scoped sessions of a per-session clamped actual
duration:

| Session shape | Used contribution |
|---|---|
| `status = ABSENT` | 0 (D-03 -- this zero IS the waste signal) |
| `actual_start IS NULL` | 0 (nobody ever checked in) |
| `actual_start` set, `actual_end` set | `min(actual_end, scheduled_end) - max(actual_start, scheduled_start)`, floored at 0 |
| `actual_start` set, `actual_end IS NULL`, `status = ACTIVE` | EXCLUDED from booked AND used; counted in `in_flight` (D-09) |
| `actual_start` set, `actual_end IS NULL`, any other status | 0 used, booked still counts (a finished session with no end stamp is not evidence of use) |

Two judgement calls being made explicitly rather than silently:

1. **The used interval is clamped to the scheduled window.** D-03 fixes the
   early-end case (`actual_end - actual_start`) and the clamp preserves it
   exactly, because `actual_end < scheduled_end` there. The clamp only bites on
   overrun, which D-03 does not address. It is needed because `seed_term` sets
   `actual_start` as early as 4 minutes BEFORE `scheduled_start`
   (`seed_term.py:368`, delay `randint(-4, 9)`); unclamped, a room could report
   more used hours than it had booked and the rate could exceed 100%, which is
   meaningless for a capacity metric.
2. **In-flight sessions are excluded from both sides, not clamped to now.** The
   module contract forbids a baked-in `timezone.now()` (`reporting.py:6-8`), and
   a class still running has no final used-hours -- counting its booked hours
   against an incomplete numerator would fabricate waste that has not happened.
   Excluding it from both sides keeps the ratio honest. `RoomUtilization.in_flight`
   surfaces the count so the exclusion is visible in the UI and can never read as
   a silent zero, which is exactly what D-09 forbids.

**AVAILABLE seconds** (the denominator, D-01 as corrected by D-06):

```
available = physical_room_count
          * sum over each calendar date d in [start, min(end, as_of)]
                where d.weekday() is in teaching_weekdays(term)
            of sum over each block b in campus_block_ladder(term)
                of b.duration_seconds
```

Every factor is derived. `physical_room_count` comes from a queryset, block count
is `len(ladder)`, teaching days come from the term's own schedules. No literal
11, 5 or 7 may appear anywhere in this arithmetic. `DayOfWeek` is
`MON = 0 .. SUN = 6` (`scheduling/models.py:12-19`), which is byte-identical to
Python's `date.weekday()`, so the two map with no translation table -- assert
this in a test rather than trusting it.

**WASTED seconds** = `absent_seconds + unused_held_seconds`, where
`absent_seconds` is the booked span of ABSENT sessions and `unused_held_seconds`
is `booked - used` over held sessions (this captures a late start AND an early
release; do not key it off the `ended_early` flag -- see the trap below).

**RATES.** `utilization_pct = used / available`. `booking_pct = booked / available`.
Both through `_hours_pct`, never float division.
</metric_contract>

<traps>
Named failure modes for this plan. Each has bitten this project or is provable
from the code already read.

- **`ended_early` is NOT the early-release signal in real data.**
  `grep -n "ended_early" scheduling/management/commands/seed_term.py` returns
  nothing: the seeded term never sets the flag, but it DOES shave 5 or 12 minutes
  off `actual_end` on roughly 40% of held sessions (`seed_term.py:370-371`). A
  waste metric keyed off the boolean reports exactly zero reclaimable hours
  against the only dataset anyone can check it with. Derive waste from the
  timestamps; the flag may be reported as a supporting count, never as the
  arithmetic basis.
- **`Room.is_virtual` is a property, not a column** (`campus/models.py:44-54`).
  It cannot appear in `filter()`, `Q()`, `values()`, `order_by()` or
  `Count(filter=...)`; any queryset reaching for it raises `FieldError`. Use the
  in-production idiom from `seed_term.py:253-255`.
- **Import direction.** `scheduling/` imports from `web/` in exactly zero places
  today (verified). The ladder therefore lives in `scheduling/reporting.py` and
  `web/room_state.py` imports it, not the reverse. This is a deliberate deviation
  from the PATTERNS.md recommendation, which sited the helper in
  `web/room_state.py`; that placement would have made `scheduling.reporting`
  the first module in the app layer to depend on the view layer. PATTERNS' actual
  goal -- one derivation, two consumers -- is met either way.
- **A None ladder is legitimate.** `room_timetable` returns `None` when the term
  is None or has no ACTIVE schedules (`room_state.py:108-115`). Both callers must
  keep handling it, and a None ladder means a zero denominator, which `_hours_pct`
  must absorb without raising.
- **MSSQL.** Never build a `pk__in` from an unbounded batch (the 2100-param limit
  that broke `reset_term`). Materialize with `list()` before any follow-up query
  inside an `.iterator()` loop (HY010).
</traps>
