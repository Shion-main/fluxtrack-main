{% comment %}IFO-09 tier T2: the campus day-by-block utilization heat grid. Deliberately built on the SAME `.tt` skeleton as the printed room timetable (ifo/room_detail.html), driven by the SAME derived block ladder. An IFO admin reading this grid and a room's printed timetable in two tabs must never have to reconcile two different grids -- if the rows ever stop matching, the shared derivation has been bypassed somewhere. Colour is an ACCELERANT, never the message. Every cell prints its percentage as visible text, so a reader in greyscale, on a mono printer, or with any colour vision deficiency gets the complete answer from the number alone. The shading is RELATIVE to the busiest cell in this grid, not an absolute good/bad ladder. No decision in this phase sets a utilization target and the campus baseline is ~26%, so an absolute ramp would render permanently red in its normal state. The legend says so in words, because a scale whose meaning is not stated is a scale a reader will misread. A cell the term does not timetable carries NO capacity, which is a different fact from a cell that had capacity and went unused. It uses the established free-cell idiom from room_detail.html:89 -- a dash marked aria-hidden paired with an sr-only word -- rather than a misleading 0%. ASCII-only.{% endcomment %}
