# 2026-07-07 — Phase 04.1: Real-Data Integration (full 2T SY2025-26 term load)

Started from a non-technical question ("what does a faculty user do?") and turned into
loading the **real** MMCM 2nd Term SY 2025-2026 registrar export into FluxTrack as the live,
checkable term — then finalizing it and logging one real-world attendance gap the data exposed.

## What shipped

**Phase 04.1 — Real-Data Integration (INSERTED, Complete 2026-07-07).** 4 plans / 3 waves,
executed against MSSQL LocalDB in the main checkout (worktrees off — see gotcha below).

- **04.1-01** — a stdlib-only `.xlsx` reader (`scheduling/xlsx.py`, zipfile + xml.etree, no
  openpyxl/pandas) + pure helpers (`scheduling/importing.py`): `classify_room` (prefix→building
  map), `parse_meetings`, `modality_for_room`, `normalize_name_key`, and `reconcile()` (the
  four-bucket partition). 26 DB-free unit tests.
- **04.1-02** — `load_room_master` (114 named rooms + real capacities from the 116-tab room-master
  xlsx; tab name is the authoritative code, C4=name, O4=capacity) and a reversible `reset_term`
  behind a `--yes` guard.
- **04.1-03** — hardened `import_offerings`: reads xlsx by default (csv path + `--building/--floor`
  filters preserved), keeps online/gym meetings (removed the skip-V and the GYM2-drop bug), stamps
  per-meeting modality by room, dedups instructors, routes roomless-physical sections to a flagged
  "TBA" room, and prints a reconciliation report. Retired the dead `R3ParityTests`.
- **04.1-04** — ran the live clean-load (reset → room-master → import → materialize --days 14),

## Locked decisions (D1–D10, in 04.1-CONTEXT.md)

xlsx sources read directly (no new dep); explicit prefix→building map **R→Academic, A→Admin,
GYM→Gym, V→Online(virtual), P/U/typo→Unassigned (flagged)**; **per-meeting modality by room**
(a blended course's physical days are scannable, its V-room days are online/checker-verified);
term **anchored to today** (not the historical 2T dates) so check-in is live; instructors deduped
**by email then normalized name** (57 blank-email rows = ~10 real people, e.g. VILLANUEVA
comma-variant merges → 200 total, ~10 email-less flagged as unable to Entra-auth); the 44
roomless-physical sections load against a shared **TBA** placeholder + flagged (section labels
like A151/C110 are NOT rooms); reversible clean-load that never mixes with the seed_demo term.

## Live outcome

Reconciliation balanced exactly: **1042 intact + 44 roomless-TBA + 14 online-no-room + 111
no-schedule = 1211**, **2021 meetings**. Live DB: **2,113 schedules · 4,230 sessions · 200 faculty
· 218 rooms · 5 term buildings**. Flagged (never silently dropped): typo rooms 404/516, Unassigned
U101/U102, 10 email-less instructors.

## Data reality learned (verified against the real files)

- 1,211 offerings; 168 distinct rooms in the schedule (92 virtual V + 76 physical); a separate
  **114-room master** (SY 2023-24 workbook, term-independent room code/name/capacity) added real
  names ("ECE Laboratory 2", "Global Classroom", "Multifunction Room") + capacities.
- More than half the catalog is Online — the old importer silently dropped **1,215 of 2,021
  meetings** (every V-room + gym). Old: 483 sections / 806 schedules / 145 faculty → new: 1,100 /
  2,113 / 200.
- The registrar's `Schedule` column is compound free-text (`DAY [START-END] ROOM`, comma-sep).

## Real bugs found by real data

- **reset_term 2100-param wall (fixed):** deleting ~2,113 sessions in one `id__in=[...]` — and the
  deletion collector's `SET_NULL` update on `Session.handover_from_session` — overflowed SQL
  Server's 2100-parameter-per-statement limit (pyodbc `07002`). Passed Plan 02's tests only at
  R3-slice scale. Fixed by batching all id lists at `PARAM_CHUNK=900`.
- **GYM rooms were being dropped** by the old `parse_room` "2nd char is a digit" rule (`GYM2` fails);
  the new explicit prefix map fixes it.

## Finding logged as follow-up: Phase 04.2 — Co-Scheduled Session Attendance

Logging in as GARAY surfaced that his sections A301 (46 enrolled, A408-A) and A302 (24 enrolled,
A408-B) meet at the **same M/W 3:45 slot** in the two halves of a divisible room. He can only scan
one room, so the sibling session would be **falsely marked Absent** by the JOB-02 sweep. This is
**widespread: 129 concurrent-section slots across 54 of 200 profs (~27%)**. Not a data defect (the
import is faithful; merging would destroy distinct rosters) — it's an attendance-layer gap. Logged
as **Phase 04.2** with success criteria + design options (recommended: one scan marks the
co-scheduled group present).

## Gotchas / conventions confirmed

- **Worktrees are unsafe for this repo:** `data/raw/*.xlsx` and `.env` are gitignored, so an
  isolated worktree has neither the real data nor the DB config. Set `workflow.use_worktrees=false`;
  executors run sequentially in the main checkout. (Also avoids the shared `test_fluxtrack`
  test-DB collision on parallel Wave-2 runs.)
- App is **MSSQL-only** (no SQLite fallback); LocalDB via `.env`.
- Small UI fix along the way: the faculty "This week" list now shows start–end times (was start only).
